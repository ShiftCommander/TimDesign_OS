# System Active OS Rules

## Philosophy

TimDesign_OS is an active business operating system, not an archive.

The system should make TimDesign easier to sell, deliver, publish, and manage.

## Rules

- Keep the active OS small.
- Prefer canonical files over variants.
- Prefer business execution over meta-work.
- Do not preserve content just in case.
- Do not create permanent archive folders inside the active OS.
- Use Git history, Drive history, or a temporary external quarantine for safety.
- Keep indexes human-readable.
- Promote only durable, reusable knowledge into the OS.
- Keep live project/prospect/contact status in Notion.

## Cleanup Rule

When simplifying, merge useful fragments into canonical files and leave obsolete variants behind.

Do not copy old folders mechanically.

## Done Means

The OS is clearer, smaller, easier to use, and more directly connected to revenue, delivery, offers, prospection, content, or decision-making.
