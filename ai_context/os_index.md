# OS Index

Canonical entry point for AI tools working with TimDesign context.

## Read first

1. `ai_context/knowledge_profile.md`
2. `ai_context/agent_operating_protocol.md`
3. `ai_context/tool_registry.md`
4. `ai_context/skills_registry.md`
5. `ai_context/project_registry.md`

## Purpose

This OS gives ChatGPT, Codex, Antigravity, OpenCode, Gemini, Cursor, and related tools a shared context layer.

The goal is to reduce repeated explanations, repeated setup attempts, and repeated manual handoffs.

## Default behavior

- Read the relevant OS files before asking for context.
- Reuse documented successful workflows.
- Record durable new findings after successful work.
- Keep updates compact and practical.

## Current canonical files

- `knowledge_profile.md`: what the assistant should assume Timtim already knows.
- `agent_operating_protocol.md`: how agents should behave when working from this OS.
- `tool_registry.md`: tools, connectors, apps, MCPs, setup notes, and known fallbacks.
- `skills_registry.md`: reusable skills and when to use them.
- `project_registry.md`: active and historical projects with canonical context pointers.

## Maintenance rule

When a workflow, tool, skill, or project state becomes reusable, update the matching registry.
